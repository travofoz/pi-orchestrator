Target platform: unbranded ESP32-C3 SuperMini (single-core RISC-V 160 MHz, 400 KB SRAM, 4 MB flash). Host viewport: ~270x518 tmux on Pixel 6a AVF VM with 1.87 GB RAM — keep builds lean. Hardware constraints: no display, no toggle switches, no external reset; only GPIO9 button and GPIO8 LED. Partition layout: 2x 1.40 MB OTA slots (ota_0, ota_1) + 1.12 MB LittleFS storage. SPIFFS is banned — LittleFS mandatory. All timing must be non-blocking (millis()/vTaskDelay), no delay(). GPIO9 is a strapping pin — the 20s rollback handshake must validate HIGH after release to avoid ROM download mode lockup. mbedtls SSL buffers limited to 4096 bytes via sdkconfig. ADB crypto: SPAKE2 PAKE → mTLS 1.3, 6-digit PIN seed over elliptic curve, RSA-2048 auto-generated identity. Wireless Debugging auto-disconnects on Wi-Fi loss — the device creates an isolated zero-internet LAN to keep Android happy. Out of scope for this spec: physical case CAD, power bank integration, non-Android companion apps. OTA updates are dual-path: AsyncElegantOTA for firmware (slot flip) and LittleFS.bin upload for frontend assets. The spec is truncated at ~16KB — sections 6.2 onward (Application Controls, De-Bloat Pipeline) are partially visible; additional ADB macros beyond those listed will be added as PROGMEM entries in phase 09 without changing the architecture.
